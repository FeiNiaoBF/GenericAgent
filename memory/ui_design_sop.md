# ui_design_sop · 生成式UI/UX设计规范 (v1.0)

> 来源：`ui-ux-pro-max` 技能（SKILL.md + 10 CSV + 7 references）
> 用途：设计/审查UI时的**强制检查清单**，按优先级递减排列
> 交叉：组件开发→`vue3_component_sop` | 博客视觉→`yeekox_blog_style_format_sop`

---

## 0. 前置澄清（不可跳过）

设计前必须确认：
1. **品牌调性** — 正式/活泼/极简/奢华？未确认→用中性方案
2. **可访问性等级** — AA（默认）或 AAA？
3. **目标设备** — mobile-first → 响应式断点 640/768/1024/1280
4. **明暗模式** — 是否需要双主题？需要→从第一层token就规划

---

## 1. 颜色系统（最高优先级）

**色彩空间**：用 **OKLCH**（感知均匀，Web支持 `oklch()`）

**12阶色阶**（每个色相生成）：

```
色阶| 用途
50  | 背景/hover底
100 | 浅底色
200 | 边框/分割线
300-400 | 次要元素
500 | 默认（主色调）
600-700 | hover/active
800-900 | 深色文字/深色模式背景
950 | 最深/深色模式文字
```

**调色板比例**：中性色 60% + 主色 30% + 强调色 10%

**对比度红线（WCAG AA）**：
- 正文文字 ≥ **4.5:1**
- 大文字(≥18px bold / ≥24px) ≥ **3:1**
- 非文本元素(图标/边框) ≥ **3:1**
- 必须用工具验证，禁目测

**状态色生成**：每个语义色 ≥ 4变体
```
default → hover(深1-2档) → active(更深) → disabled(40% opacity)
focus   → 2px solid ring + offset（非默认outline）
```

**命名**：`{role}-{property}-{modifier}` 如 `color-primary-bg-hover`
**禁止**：❌ 硬编码hex/rgb → ✅ CSS变量 | ❌ 颜色作唯一信息载体

---

## 2. 字体排版

**字体栈**：主字体(sans-serif) + 等宽(monospace) + emoji，各 ≤ 3个回退

**模块化缩放** (rem，base 16px)：
```
xs: 0.875rem(14px) | sm: 1rem(16px) | base: 1.125rem(18px)
lg: 1.25rem(20px)  | xl: 1.5rem(24px)| 2xl: 2rem(32px)  | 3xl: 3rem(48px)
```

**关键规则**：
- 正文 ≥ 16px，移动端 ≥ 14px
- 行高 1.4-1.8（正文1.6最优）
- 段落间距 = 行高 × 1.5
- **必须用 rem**，禁用 px 定义字体大小（破坏浏览器缩放）

---

## 3. 间距与布局

**基础网格**：**4px** 所有间距为4的倍数

**间距比例尺** (rem)：
```
1: 0.25(4px) | 2: 0.5(8px) | 3: 0.75(12px) | 4: 1(16px)
5: 1.25(20px)| 6: 1.5(24px)| 8: 2(32px)    | 10: 2.5(40px)
12: 3(48px)  | 16: 4(64px) | 20: 5(80px)   | 24: 6(96px)
```

**亲密性原则**：相关元素间距 < 不相关元素间距（视觉分组靠间距）

**布局**：页面级 Grid + 组件级 Flexbox | 列数 4/8/12 | mobile-first min-width

**禁止**：❌ 奇数间距 | ❌ padding/margin 混用无规则

---

## 4. 组件设计

**状态矩阵**（每个组件必须覆盖）：
```
default | hover | active | focus | disabled | loading | error
```

**尺寸系统**：sm / md / lg（全局统一，禁自造尺寸）

**变体层级**：`fill`(强调) > `outline`(次要) > `ghost`(极简)

**触摸目标**：≥ **44×44px**（WCAG 2.5.5）

**圆角**：用token统一（sm/md/lg/full），禁硬编码数值

**键盘导航**：Tab顺序合理 + Enter/Space触发 + Escape关闭

---

## 5. 可访问性（贯穿全流程）

**必检清单**：
- [ ] 对比度达标（正文4.5:1 / 大文本3:1 / 非文本3:1）
- [ ] 所有图片有 `alt`（有意义）或 `alt=""`（装饰图）
- [ ] 表单有 `<label>` + `aria-describedby` 错误提示
- [ ] 焦点样式可见（禁 `outline: none` 无替代）
- [ ] 动画支持 `prefers-reduced-motion`
- [ ] 禁用元素说明原因（不单靠 opacity+cursor）
- [ ] 颜色不是唯一信息载体（辅以图标/文字/形状）

---

## 6. 设计Token架构

**三层**：
```
Layer 1 原始值：--blue-500: oklch(65% 0.24 250)
    ↓
Layer 2 语义层：--color-primary: var(--blue-500)
    ↓
Layer 3 组件层：--button-bg: var(--color-primary)
```

**主题切换**：CSS变量 + class切换（`html.dark`），深色模式用独立色阶（非简单反色）

---

## 7. 响应式

mobile-first，断点：640 / 768 / 1024 / 1280
网格：1列 → 2列 → 3-4列 → 6列 → 12列

---

## 8. 代码质量

- 组件 ≤ 300行 | Props ≤ 10个
- 禁内联样式（用 className/token）
- TypeScript类型 + Storybook故事 + 测试文件

---

## 9. 反模式速查

| ❌ 错误 | ✅ 正确 | 原因 |
|---------|---------|------|
| 硬编码 `#hex` | CSS变量 | 主题无法切换 |
| px 字体 | rem 字体 | 破坏浏览器缩放 |
| disabled只改opacity | opacity + 原因提示 | a11y不达标 |
| 深色模式反色 | 独立暗色色阶 | 对比度失控 |
| `outline: none` | 自定义focus样式 | 键盘用户无法导航 |
| 只用颜色传达状态 | 颜色+图标+文字 | 色盲用户无法区分 |

---

## 10. 核心铁律（不可违反）

1. **严格按11阶段顺序**，不可跳步
2. 颜色用 OKLCH + 12阶色阶 + 语义命名
3. 字体 rem + 行高 1.4-1.8
4. 间距 4px 基础网格
5. 组件覆盖7种状态 + 44px触摸目标
6. 对比度必须工具验证 ≥ WCAG AA
7. 颜色禁作唯一信息载体
8. Token三层架构（原始→语义→组件）
9. 深色模式独立色阶，非反色
10. 每个可交互元素必须有可见焦点样式
