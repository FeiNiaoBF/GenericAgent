# Autonomous Quality Filter

> **位置**: memory/autonomous_operation_sop/filter_rules.md
> **触发**: 任务规划Step3 (设计TODO) 和 Step5 (执行前) 必须过此过滤器
> **设计于**: 2026-04-29 | 基于R10-R17历史模式分析

## 一、核心问题

R10-R17中，90%的自主行动报告结果为"0 issues/通过/健康"。这些任务消耗回合却无产出——因任务设计阶段未做质量过滤。

## 二、三级过滤器

### L1: 阻断词过滤器（规划阶段）
任务标题/描述命中以下关键词且**无**具体假设 → **直接拒绝**，不进入TODO：

| 关键词 | 判定逻辑 |
|--------|----------|
| 健康检查 | 除非指定"上次X时发现Y问题，现复查" |
| 过期清理 | 除非指定"预期至少N个过期文件/条目" |
| 例行扫描 | 除非指定"上次扫描于[日期]，预期新增M项" |
| 验证/核实 by itself | 除非紧跟具体假设：验证"X是否导致Y" |

**格式要求**：任何"扫描/检查/审查"类任务必须包含(Hypothesis, Expected Finding)对。

### L2: 零基线过滤器（执行前）
执行前必须回答：
1. **上次同类检查结果是什么？**（查history.txt）
2. **自上次以来系统发生了什么变化？**（若未变化且上次0发现 → 跳过）
3. **如果结果仍是"0问题"，这个任务有价值吗？**（若不是 → 转为假设驱动）

### L3: 重复过滤器（连续两轮之间）
- 上次执行过同类检查 → 间隔至少5轮才能再次执行
- 上次结果为0发现 → 间隔至少10轮，除非有证据表明系统状态已变更

## 三、转换规则

不满足过滤器的任务**不可直接删除，应转换**：

| 原始任务 | 转换后 |
|----------|--------|
| "扫描X检查问题" | "验证假设：上次发现的A缺陷是否已扩散" |
| "清理Y过期文件" | "若存在Z天未更新文件 → 归档，否则记录'无清理必要'" |
| "检查Z健康状况" | "对比Z的N项指标与M天前的基线，标记偏离>10%项" |

## 四、集成方式

1. `task_planning.md` Step3 设计TODO时：每个任务满足L1方可进入候选
2. 执行前：过L2零基线检查
3. 子任务选择时：过L3重复检查

## 五、验收标准
- 下3轮自主行动中，"0发现"报告占比 < 20%（当前90%）
- 被过滤任务有明确的转换记录（不可悄无声息丢弃）
