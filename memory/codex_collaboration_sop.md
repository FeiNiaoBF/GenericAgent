## 执行摘要
① 判断协作阶段(计划/诊断/复现/执行/复查)→② 按阶段范式执行 → 🛑 以边界换聚焦，以信息换准确，以文档换稳定

## 核心三原则
1. **边界换聚焦** — 明确不做什么/不可妥协，结果才聚焦
2. **信息换准确** — 目标/背景/约束说清，返工越少
3. **文档换稳定** — 共识尽早落文档+测试闭环

## 最小闭环骨架
```text
目标：解决【问题】；逼近【可用状态】
模式：直接推进/先诊断/先复现/先收敛计划
约束：只解决【A】，不扩展【B】；优先长期可维护/短期可交付
输出：先给判断→再落地
验证：测试/最小步骤/验收标准
```

## 12种局面速查
| # | 局面 | 打法 |
|---|------|------|
| 1 | 局面未明 | 列假设+缺失信息，按优先级输出 |
| 2 | 主线清晰 | 读上下文→动手，非关键不问 |
| 3 | 长期建设 | 优先可维护；短期方案标注风险 |
| 4 | 短期止血 | 优先可用；临时方案标治理项 |
| 5 | 不需兼容 | 硬切：不保留兼容层 |
| 6 | 连续失败 | 复盘：目标/路径/细节哪错了→停低收益 |
| 7 | 链路越深 | 评估停点→继续/收口/换路 |
| 8 | 临近收口 | 补验收+验证步骤 |
| 9 | Bug | 先复现→修→回归 |
| 10 | 多轮协作 | 落文档：目标/约束/共识/待决/下一步 |
| 11 | 任务复杂 | 收敛计划：梳理目标/约束/边界/路径 |
| 12 | 技术债 | 挂账不扩散：阻塞→说明原因；不阻塞→记清单 |

## 四个即用组合
- **方案讨论型**：收敛方案→按"问题/选项/利弊/推荐"格式
- **直接开发型**：任务明确→先判断+推荐→改完补测试
- **Bug修复型**：复现→根因→修复→回归验证
- **独立复查型**：不重复主流程→列出"必须改"+"可选优化"

## 补充句(按需追加)
- `只解决核心，不顺手扩展`
- `不考虑兼容，直接目标状态修改`
- `问题只提最关键的1~3个，按"问题/选项/利弊/推荐"格式`
- `兜底方案先说明更优正解为什么暂不可行`

## 五阶段协作流
```
计划→诊断→复现→执行→复查
模糊  未明  Bug  清晰  完成
```
触发回退条件：偏航/反复失败/需重新收敛。
