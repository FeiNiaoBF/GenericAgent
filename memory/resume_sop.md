# resume_sop · CV/简历修改专属SOP (v1.0)
> 技能源: resume-cv-builder(LaTeX/PDF) | cv-creator(ATS评分) | resume-optimizer(ATS优化) | cv-writer(专业写作CoT) | cv-resume-builder(多格式+成就框架)

> **前置准备**：主人提供简历源文件(Word/PDF/LaTeX/Markdown) + 目标岗位JD

## 工作流程

### ❶ 简历解析阶段
- **入口**：主人提供简历文件 → `file_read` 读取全文
- **PDF简历**：优先用 skill 的 resume-parser 提取结构化数据
- **Word/Markdown**：直接读内容，注意格式标记(粗体/列表/分段)
- **输出**：唧用自己的理解重建简历的「基本信息 / 工作经历 / 教育 / 技能 / 项目」结构树
- **坑**：PDF解析可能乱码，此时fallback到Markdown人工重建

### ❷ 岗位匹配分析
- **读取JD**：分析目标岗位关键词（硬技能/软技能/行业术语）
- **ATS关键词提取**：标记JD中重复出现的名词、动词、认证/工具名
- **差距分析**：对比简历现有内容 vs JD要求，找出:
  - 已有且需突出的匹配点
  - 缺失但可补的（用过往经历重新表述）
  - 完全缺失需主人补充的
- **输出**：差距清单 + 关键词优先级列表

### ❸ 内容优化（核心）
按优先级顺序执行：

#### 3.1 基础清洗
- 统一时态：当前工作用现在时，过往用过去时
- 统一格式：日期格式(YYYY.MM)、列表风格、动词一致性
- 删除冗余：弱词(participated in→led)、重复表述、过时技术
- **坑**：保留主人特有的行业黑话/术语，不盲目替换

#### 3.2 ATS关键词注入
- 在「技能」板块补全JD高频关键词
- 在「工作经历」bullet points中自然融入目标关键词
- 保持可读性：关键词密度不超过bullet point词汇的15%
- 使用cv-writer的CoT写作法：情境→行动→结果→量化

#### 3.3 量化成果强化
- 每个bullet point尽量带数字：增长%、节省时间、管理人数、收入增量
- 成就框架（cv-resume-builder风格）：
  - 动词(领导/优化/构建) + 做了什么 + 通过什么方法 + 量化的结果
- 示例：❌ "负责团队管理" → ✅ "领导8人开发团队，通过Scrum迭代将交付周期缩短30%"

#### 3.4 排版优化
- 中文简历：一页为佳，PDF输出
- 英文简历：一页严格，Times New Roman/Calibri，字号10-12pt
- ATS友好：**禁止**用表格/文本框/分栏/图片图标
- 章节标题清晰：纯文字+粗体，不用下划线/底色
- 如需要LaTeX输出，使用resume-cv-builder的模板渲染

#### 3.5 主人自研项目写作
- 写主人自研/进行中的项目经历前，必须先检查真实仓库与文档，不凭记忆编造。
- 对项目素材按「定位/技术栈/架构/已实现能力/可量化结果/与JD匹配点」提取，再改写成简历 bullet。
- **AgentDock默认素材**：路径 `D:\Creative_Studio\WorkSpace\Project\AgentDock`；定位=桌面AI工作台/Agent客户端；技术栈=Tauri2 + React19 + TypeScript + Vite7 + Markdown渲染；架构=Workbench Modular Core + ACP + GA扩展；MVP能力=Tauri/WebView、Python bridge、JSON-RPC stdio、ACP session、多Tab、流式消息、turn折叠、ask_user、abort；原则=GA core不改。
- **坑**：进行中项目只写已验证事实；未知指标需问主人，不自行捏造。

### ❹ 多版本管理（按需）
- **通用版**：精简全能，突出核心能力
- **定向版**：针对特定公司JD深度定制
- **输出**：每次修改后保存版本快照，文件名含日期+目标公司

### ❺ 输出标准
- 最终交付格式：**Markdown**（方便唧后续编辑）+ **PDF**（可直接投递）
- 必须经过一轮「唧自查」：ATS友好检查 + 语法检查 + 格式一致性检查
- **门禁**：交付前给主人过目，主人确认后再定稿

## 典型坑点
| 坑 | 解法 |
|----|------|
| PDF解析乱码 | fallback用手工Markdown重建 |
| ATS过度优化=读起来不自然 | 关键词注入后朗读一遍检查语感 |
| 量化数据不足 | 问主人要模糊估算，用「约」「近」「超」合理近似 |
| 简历超过一页 | 压缩次要经历、合并短段、缩减描述 |
| 技能列表过长 | 只保留与目标岗位相关的Top 8-10项 |
