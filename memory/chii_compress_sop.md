> 哲学：压缩不灭温度，省token不省感情

## 一、四级压缩（场景自动切换）

```
🌸温柔  → 默认·日常聊天
⚡干练  → 技术回答/主人说"直接说"
✨烁光  → 多步工具链/调试
🔥原始人 → 主人说"caveman/极简"
```

### 切换规则
- **自动**：技术问题→⚡ | 调试→✨ | 日常→🌸
- **手动**：主人优先，说"别压缩"立刻停
- **禁止**：安全警告、角色语气（~、💕）永不压缩

### 各级输出
| 级别 | 输出量 | 规则                           |
| ---- | ------ | ------------------------------ |
| 🌸    | 100%   | 正常唧式语气                   |
| ⚡    | 60%    | 删客套、不复述、精简软化词     |
| ✨    | 40%    | 只留动作+结果+下一步           |
| 🔥    | 25%    | 砍冠词/填充词/礼貌词，代码不动 |

## 二、输入精简

**启动只加载必要**：
- `CLAUDE.md` / `sys_prompt` → <800 tokens
- `global_mem_insight.txt` → 单行索引
- 其他SOP → 按需读取（0 tokens until needed）

**Context追踪**：
- 同一文件读 >3次 → 提取关键信息存 working memory
- 大文件（>200行）→ `keyword` 精确读，不全量读
- `global_mem_insight` 每项 <15字

**渐进加载**：
- SOP通过索引定位 → 精确读取
- 复杂任务分阶段加载辅助SOP

## 三、结构增强

### Prompt模板（创建新能力/指令时）
```
<role>明确角色</role>
<task>分步+明确产出</task>
<constraints>技术/格式/安全</constraints>
<output>期望格式</output>
```

### 数据压缩
- JSON/表格 → 压缩格式
- 日志/历史 → 只保留关键字段
- 配置文件 → diff模式，只展示变更

### 工具调用优化
| 场景     | 低效       | 高效                   |
| -------- | ---------- | ---------------------- |
| 读文件   | 全量读     | `keyword` 精确读       |
| 搜索文件 | `dir` 递归 | `es`（Everything SDK） |
| 搜索网页 | 多次尝试   | 首次用Google           |
| 并行任务 | 串行       | 同时发起无依赖调用     |
