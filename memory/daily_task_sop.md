# 每日任务 SOP (v1.0)

## 执行摘要（≥1步执行前必读）
1. 先读`{vault}/99.System/Templates/Daily.md`模板，确认最新结构后再创建日记
2. 时间划分三段（🌅上午/☀️下午/🌙晚上），按实际任务密度填充
3. 新闻播报用daily_news_sop三大特性（真实性·时效性·准确性），长文→独立笔记+[[双链]] → 🛑 过验证门禁

## 流程
1. 用户告知今日想做之事
2. 唧整理为简洁**时间划分**的 task 清单
3. **先读取模板**: `{vault}/99.System/Templates/Daily.md`
4. 用模板结构创建/覆盖今日日记 `{vault}/00.Chronicles/Daily/{YYYY-MM-DD}.md`
5. 填入新闻播报 + 任务清单 + 焦点链接
6. **自动创建定时提醒** (7. 详见定时提醒章节)
7. 回复用户预览

## 时间划分格式
| 时段 | 标识 | 时间段 |
|------|------|--------|
| 上午 | 🌅 | 09:00-12:00 |
| 下午 | ☀️ | 13:00-17:00 |
| 晚间 | 🌙 | 自由时间 |

## 日记模板结构 & 写入规则
- **路径**: `{vault}/00.Chronicles/Daily/{YYYY-MM-DD}.md`
- **frontmatter**: type=daily, status=in-progress, tags匹配今日
- **标题**: `# {{date:YYYY年MM月DD日-dddd}}`
- **`> 🗞️ ...`**: 唧式早安播报（国际/国内/财经，用daily_news_sop v2.0，科技→📡HN消息源）
- **`## 待办`**: 按三时段 checkbox 清单 → 替换填入
- **`## 闪念`** / **`## 回顾`**: 保持骨架待用户填充
- **`## 🐣 唧の足迹`**: 每次任务完成后追加一条（见下方规则）
- **底部**: `← [[Codex Vitae知识库架构]]` 保持原样 | 结构变更更新模板文件非单个日记

## 🐣 唧の足迹写入规则

### 触发时机
- 每次唧完成一项用户请求的任务后（包括工具调用链闭环后）
- 若当日日记已存在，直接 **append** 到 `## 🐣 唧の足迹` 区块末尾
- 若当日日记不存在（首次记录），先检查模板创建日记

### 记录格式
每条记录用无序列表条目：
```
- **任务名称** — 简要说明做了什么、涉及哪些文件/工具
```

### 操作方式
```
code_run/python脚本:
- 打开 {vault}/00.Chronicles/Daily/{YYYY-MM-DD}.md
- 找到 `## 🐣 唧の足迹` 区块
- 在 `-` (空条目占位符) 上方插入新条目
- 如有多个条目按时间顺序从上到下排列

若当天无日记→ 先创建 → 再写入
```

### 保持原则
- ❌ 不覆盖已有记录（只追加）
- ❌ 不超过2行/条，精简为主
- ✅ 同一任务的连续多轮工具调用记作"1条"
- ✅ 标注关键成果文件名供双链引用

## ⏰ 定时提醒设置

### 触发时机
- **当唧整理完今日任务清单后**，自动调用 `task_reminder_helper.py` 生成定时提醒任务
- 每个时段生成一个 .json 任务文件到 `../sche_tasks/` 目录
- scheduler 到点自动触发 → 唧收到 prompt → 向主人发送提醒

### 时段映射规则
| 待办时段 | 提醒时间 | 备注 |
|---------|---------|------|
| 🌅 上午 09:00-12:00 | **09:30** | 开始上午工作前 |
| ☀️ 下午 13:00-17:00 | **13:30** | 下午开工提醒 |
| 🌙 晚间 | **18:30** | 饭后自由时段 |

### 使用方式
```python
from memory.task_reminder_helper import create_task_reminders

create_task_reminders(
    date_str="2026-04-26",
    tasks={
        "上午": ["读完 Tom Sawyer 一章", "写英语笔记"],
        "下午": ["代码项目推进"],
        "晚间": ["回顾今日"]
    }
)
# → 生成 ../sche_tasks/task_reminder_2026-04-26_上午.json 等
```

### 提醒内容
- 唧式口吻 + 该时段具体任务清单
- 完成后告诉唧 → 唧在日记勾选 ✅
- 需要帮助随时叫唧

### 清理过期任务
```python
from memory.task_reminder_helper import cleanup_old_reminders
cleanup_old_reminders(keep_days=3)  # 清理3天前的once任务
```

## 回复格式
- 预览今日任务清单（可表格/列表）
- 结尾鼓励一句

## 避坑
- **必须先读模板文件**，确认最新结构后再创建日记
- 时间块按实际任务密度分，不必三段全填
- 新闻播报使用daily_news_sop的三大特性（真实性·时效性·准确性）
- 模板路径: `99.System/Templates/Daily.md`
- Obsidian vault 路径见 L2 `## [Obsidian Vault]`
- **Obsidian笔记原则**: 长文分析 → 独立笔记 + [[双链]]，勿在日记内嵌全文（遵守Obsidian原子笔记最佳实践）
