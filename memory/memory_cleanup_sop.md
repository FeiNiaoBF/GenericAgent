# 记忆整理 SOP

## 核心原则：存在性编码
LLM自身是压缩器+解码器。L1只需让它**意识到某类知识存在**，它就能通过tool call自行取用深层内容。

**L1本质：用最短词数表达——什么场景下有什么记忆可用（存在性）。**

L1两类内容，统一ROI评估：
1. **SOP路由**：`关键词:SOP名`
   判断标准：模型运行时能看到这个文件 → 节省多次试错 → ROI极高
2. **关键事实**：环境变量/全局状态/用户偏好（环境封闭，一步试错）
   判断标准：不能靠推理得出的环境事实 → 记住即省5+轮 → ROI高

## 垃圾清理规则
❌ **禁止人L1**：临时任务进度、中间产物路径、可推理内容、冗余条目（同义不同名）
❌ **禁止人L2/L3**：SOP子文件（模型不自动读L3，列了也没用）

## 双向同步
L1变更时不必改L2（会自动滞后），但L2/L3变更后L1必须补上路由。
L2修改前必须先读META-SOP §validate验证有效性+反垃圾。

## L4管理
L4_raw_sessions 只存"30轮以上且有长期价值"的对话原始记录（成本：disk），其他碎片化内容不做L4

## 触发清理
- 当 L1 > 50 行 或 L2 > 200 行 时主动清理
- 或用户说 "清理记忆"
