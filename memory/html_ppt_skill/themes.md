## 主题 DNA（生成页面必须吸收，不可只换色）

| Theme | 明暗基调 | 视觉 DNA | 布局/组件要求 |
|---|---|---|---|
| `engineering-whiteprint` | 浅色优先 | 白底坐标纸/工程蓝灰线、细网格、模块编号、轻阴影、技术注释感 | 卡片像白板模块；可视化用线框、标尺、编号节点；适合默认 CS 教学 |
| `solarized-light` | 浅色护眼 | 米黄色低眩光背景、柔和青蓝/橙色强调、纸张阅读感、低对比阴影 | 减少强渐变；正文区宽松；适合长课和理论解释 |
| `academic-paper` | 浅色正式 | 论文纸张、衬线/严肃标题、引用块、脚注感、表格线 | 布局像讲义/论文摘要；强调定义、定理、例子、证明/推导 |
| `catppuccin-latte` | 浅色现代 | 柔和奶油底、圆角卡片、粉紫/蓝绿 accent、舒适开发者气质 | 卡片更圆润；状态标签柔和；适合现代编程课 |
| `tokyo-night` | 深色优先 | 深蓝黑背景、霓虹蓝紫/青绿高亮、代码编辑器氛围、强代码块 | 首屏与代码区可更戏剧化；正文保持克制高对比 |
| `blueprint` | 深色优先 | 深蓝图纸背景、线框模块、连接线、坐标/网格、编号节点 | 适合架构/网络/流程；必须突出连接关系与方向箭头 |
| `corporate-clean` | 浅色正式 | 企业白/灰、克制蓝色、清晰 KPI 卡、表格、流程编号 | 少装饰；强调结构、对比、决策、可交付感 |
| `nord` | 冷色浅/深均可 | 北欧冷灰蓝、低饱和、冷静云基础设施感 | 卡片边界清楚但不炫；适合网络、云、系统主题 |
| `minimal-white` | 浅色极简 | 大留白、黑灰文字、少量单色 accent、无多余装饰 | 通过字号/间距/细线建立层级；适合内部评审和内容优先 |
| `catppuccin-mocha` | 深色护眼 | 暖深色背景、柔和粉紫/桃/蓝 accent、舒适夜间开发者感 | 深色卡片要分层；代码区与正文区用色统一但高可读 |

## 主题守恒规则

1. 选定主题后，至少落实 5 类差异：`背景纹理`、`字体气质`、`卡片形态`、`边框/阴影`、`可视化线条/节点`。
2. 不能只修改 `--bg` / `--accent` 颜色；若多个主题预览在截图缩略图中结构相同，判定失败。
3. 深色主题默认展示深色变量，必须补完整浅色变量；浅色主题默认展示浅色变量，必须补完整深色变量。
4. 不适合长正文的强视觉主题（如 `cyberpunk-neon`、`vaporwave`、`y2k-chrome`）只用于封面/专题/短页，不作为 CS 长教学默认。
